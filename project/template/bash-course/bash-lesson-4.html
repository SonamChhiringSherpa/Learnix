Understanding File Permissions

Every file and directory in a Linux system has a set of permissions that define who can read, write, or execute it. These permissions are shown when you use ls -l:

$ ls -l script.sh -rw-r--r-- 1 user user 0 Jan 1 00:00 script.sh

This means:

  • - – It’s a regular file (not a directory).
  • rw- – The owner (user) can read and write.
  • r-- – The group can only read.
  • r-- – Others (everyone else) can only read.

To make a script executable:

$ chmod +x script.sh $ ls -l script.sh -rwxr-xr-x 1 user user 0 Jan 1 00:00 script.sh

Now the owner, group, and others can all execute the file.

Octal Notation for Permissions

Bash also supports setting permissions using numbers. Each permission is represented by a digit:

  • r = 4 (read)
  • w = 2 (write)
  • x = 1 (execute)

These are added together for each of the three groups: user, group, and others.

$ chmod 755 script.sh # 7 = rwx (4+2+1), 5 = r-x (4+0+1), 5 = r-x $ ls -l script.sh -rwxr-xr-x 1 user user 0 Jan 1 00:00 script.sh

Another example:

$ chmod 644 notes.txt # 6 = rw-, 4 = r--, 4 = r-- $ ls -l notes.txt -rw-r--r-- 1 user user 14 Jan 1 00:00 notes.txt

Changing Ownership

Only the file's owner or the superuser can change ownership of a file or directory using chown:

$ sudo chown newuser notes.txt $ ls -l notes.txt -rw-r--r-- 1 newuser user 14 Jan 1 00:00 notes.txt

Practice Tip

Try creating files and applying different permission combinations using both symbolic and numeric methods:

  • Create a file and restrict access only to yourself
  • Make a script readable and executable for everyone
  • Use chmod and chown together to secure shared directories
