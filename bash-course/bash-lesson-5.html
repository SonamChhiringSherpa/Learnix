Your First Script

Bash scripts let you automate tasks by writing sequences of commands in a file. Every script should begin with a shebang (#!/usr/bin/env bash) to tell the system which interpreter to use.

After writing the script, use chmod +x to make it executable. You can also pass arguments to scripts to make them dynamic.

$ cat hello.sh
#!/usr/bin/env bash name=$1 echo "Hello, $name!"
$ chmod +x hello.sh< $ ./hello.sh Learner Hello, Learner!

Variables in Bash: Storing and Using Data

Variables are placeholders to store data like strings, numbers, or command outputs. You assign a value without spaces around the = sign, and you access the value by prefixing the variable name with $.

$ # Assign a string to a variable $ greeting="Hello, World!" $ echo $greeting Hello, World! $ # Assign the output of a command $ today=$(date +%Y-%m-%d) $ echo "Today's date is $today" Today's date is 2025-08-19

Tips:

  • Use "" around strings with spaces, e.g. name="John Doe".
  • Use ${variable} syntax to clearly delimit variable names when appending text, e.g., echo "User: ${name}123".

Conditionals: Making Decisions in Your Script

Conditionals let your script perform different actions based on tests, like comparing numbers, checking file existence, or matching strings. The syntax uses if, then, else, and fi (end if).

$ count=15 $ if [ $count -gt 10 ]; then > echo "Count is greater than 10" > else > echo "Count is 10 or less" > fi Count is greater than 10 $ # Check if a file exists $ if [ -f "notes.txt" ]; then > echo "notes.txt exists" > else > echo "notes.txt does not exist" > fi notes.txt exists

Common Conditional Tests:

  • -eq, -ne, -lt, -le, -gt, -ge: numeric comparisons (equal, not equal, less than, etc.)
  • -f filename: true if file exists and is a regular file
  • -d directoryname: true if directory exists
  • string1 = string2: true if strings are equal (use [[ ... ]] or quote strings)

Best Practices:

  • Always quote variables inside test brackets to avoid errors: if [ "$var" = "value" ]
  • Use double brackets [[ ... ]] for advanced pattern matching and safer syntax.
